**象限定义**：

| 象限 | 判定标准 |
|------|----------|
| **优先做** | 需求真实高频 + 竞品粗糙或无 + 技术可行 |
| **差异化做** | 需求真实 + 竞品已有但有缺口 + 需明确差异化角度 |
| **观察** | 需求弱或未充分验证 + 部分可行 |
| **不做** | 竞争饱和无差异化空间，或需求被证伪 |
| **排除** | 结构性技术不可行 |

**填写指引**：
- "用户需求"综合 Part 0.2 社区频率信号 + Part 2.3 场景真伪判定 + Part 4.3 步骤三校验结果
- "竞品质量"取 01_市场与竞品全景 中的竞品覆盖矩阵评级
- "技术可行性"取 Part 3.1 TCR 数据 + Part 3.3 卡点根因
- "综合判定"基于 Part 1.4 象限逻辑，经 Part 2–4 数据修正
